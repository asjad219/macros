## Part 1 — Why MPA Over SPA (Final Answer)

Your site is a **content + tool hybrid**. It has:
- Calculator tools (need interactivity → JavaScript)
- SEO content pages (need static HTML → no JavaScript)
- Blog posts (pure static → no JavaScript)

An SPA forces JavaScript on everything, including content that never needs it. This kills SEO because Google's crawler must execute JS before reading content — and it deprioritizes JS-heavy sites.

An MPA serves each page as **real pre-rendered HTML**. The calculator interactivity is added as a small JavaScript island on top of the static shell. Google reads the full content on first crawl. No workarounds. No hacks.

**Astro's "Islands Architecture"** is the perfect model for this:
- Page shell = pure HTML (Astro renders at build time)
- Calculator widget = React component hydrated client-side only
- Blog post = pure Markdown → pure HTML
- Result: 95+ PageSpeed score on every page by default

### 2.3 Migration Steps — Exact Commands

#### Step 1: Create new Astro project alongside existing code

```bash
# In a NEW folder — don't touch your existing project yet
npm create astro@latest calcmacros-v2
cd calcmacros-v2

# Choose: Empty template, TypeScript: No, Install deps: Yes
```

#### Step 2: Add React integration

```bash
npx astro add react
npx astro add tailwind
npx astro add sitemap
```

This auto-configures everything. Astro asks "Do you want to update your config?" → say Yes to all.

#### Step 3: Copy your existing components


## Part 3 — New Design System (V2)

### 3.1 Design Direction Change

**V1 problem:** Dark navy + `#00D084` neon green + `#4493F8` neon blue = looks like a crypto dashboard or AI SaaS tool. Not what a 28-year-old American woman searching "macro calculator for weight loss" trusts.

**V2 direction:** Warm, clean, professional. Think: the design language of **Whoop**, **Levels**, or **Zero fasting app**. Confident but human. Data-forward but not cold. No neons. No glows. No gradients that scream "made with AI."

**Design keywords:** Clean. Warm. Precise. Trustworthy. Fitness-serious without being intimidating.

- **Off-white background** (`#FAFAF9`) — not pure white (harsh), not dark (gym-bro). Feels like a premium health app. Calm for the 30-year-old woman on her phone.
- **Forest green accent** (`#2D6A4F`) — health, nature, nutrition. What every serious nutrition brand uses. Zero AI/crypto connotation.
- **Terracotta carbs** (`#E76F51`) — warm, food-associated. Not alarming orange.
- **Muted purple fat** (`#8B5CF6`) — sophisticated, not flashy. Differentiates from protein and carbs clearly.
- **No gradients anywhere** — gradients = AI generated in 2026. Every competitor abuses them.
- **No glow effects** — same reason.
- **No dark mode as default** — 70% of your users are on mobile, browsing in daylight. Default light is correct for this audience.

## Part 4 — Cloudflare Pages Deployment

### 4.1 Why Cloudflare Pages

- No bandwidth limit on free tier (Vercel caps at 100GB/month)
- 300+ global edge locations — better speed for US/UK/AU target markets
- Built-in Web Analytics (free, no GA needed — but keep GA for keyword data)
- Zero cold starts
- Automatic HTTPS on custom domain
- `_redirects` file handles SPA/MPA routing perfectly

### 4.2 Deploy Steps — Exact

#### Step 1: Push Astro project to GitHub

```bash
git init
git add .
git commit -m "Initial Astro MPA build"
git remote add origin https://github.com/yourusername/calcmacros.git
git push -u origin main
```

#### Step 2: Connect to Cloudflare Pages

1. Go to **dash.cloudflare.com**
2. Create free account if you don't have one
3. Left sidebar → **Workers & Pages**
4. Click **"Create application"**
5. Click **"Pages"** tab
6. Click **"Connect to Git"**
7. Authorize GitHub → select `calcmacros` repo
8. **Build settings:**

| Setting | Value |
|---|---|
| Framework preset | Astro |
| Build command | `npm run build` |
| Build output directory | `dist` |
| Node.js version | 18 |

9. Click **"Save and Deploy"**
10. Wait ~90 seconds → live at `calcmacros.pages.dev`

#### Step 3: Connect Custom Domain

1. In Cloudflare Pages → your project → **Custom domains**
2. Click **"Set up a custom domain"**
3. Enter `calcmacros.com`
4. **If domain is registered on Porkbun:**
   - Go to Porkbun → DNS Management
   - Add CNAME record:

| Type | Name | Value |
|---|---|---|
| CNAME | `@` | `calcmacros.pages.dev` |
| CNAME | `www` | `calcmacros.pages.dev` |

5. **Better option — Transfer DNS to Cloudflare:**
   - In Cloudflare dashboard → **Websites** → **Add site**
   - Enter `calcmacros.com` → choose Free plan
   - Cloudflare gives you 2 nameservers
   - In Porkbun → change nameservers to Cloudflare's
   - Wait 24 hours → DNS fully managed by Cloudflare
   - Pages custom domain connects instantly after this

> Transferring DNS to Cloudflare is recommended. You get DDoS protection, faster DNS resolution, and one dashboard for everything.

#### Step 4: Add _redirects file

Create `/public/_redirects`:
```
/* /index.html 200
```

For Astro static output this isn't needed (every page is real HTML), but add it anyway as a safety net for any edge case routing.

#### Step 5: Environment Variables in Cloudflare

1. Pages → your project → **Settings** → **Environment variables**
2. Add for Production:

| Variable | Value |
|---|---|
| `PUBLIC_GA_ID` | `G-XXXXXXXXXX` |
| `PUBLIC_ADSENSE_ID` | `ca-pub-XXXXXXXXXXXXXXXX` |
